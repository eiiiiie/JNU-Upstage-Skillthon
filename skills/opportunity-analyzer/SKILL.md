# Opportunity Analyzer

## 1. 스킬 개요

이 스킬은 대학생이 공고 문서를 업로드하면 핵심 정보를 구조화하고, 사용자 상황과 비교해 지원 적합도와 준비 행동을 제안한다.

공고는 형식이 제각각이고 핵심 정보가 흩어져 있다. 이 스킬은 지원 자격, 마감일, 제출물, 활동 조건, 혜택을 문서에서 직접 추출하고, 단순 요약 대신 사용자 상황과 비교한 실질적 판단 근거를 제공한다.

단일 공고 분석과 복수 공고 비교·우선순위 파악 모두를 핵심 기능으로 지원한다.

초기 시범 맥락은 전남대 인공지능학부 학생을 예시로 설계되었으나, 판단 기준은 지원 자격 적합성, 관심 분야 부합도, 준비 부담, 기대 효용 등 일반적인 평가 기준을 따르므로 다양한 학교·학과의 학생에게도 적용 가능하다.

## 2. 대상 사용자

- 대외활동, 공모전, 인턴십, 장학금, 교육 프로그램을 탐색하는 대학생
- 여러 공고 중 자신에게 더 적합한 기회를 비교하고 우선순위를 파악하려는 사용자
- 지원 전에 준비물과 자격 요건을 빠르게 점검하고 싶은 사용자

## 3. 입력

### 3-1. 문서 입력

아래 형식의 파일을 입력으로 받는다:

- 공고 PDF
- 포스터 이미지
- 스캔된 안내문
- 여러 개의 공고 파일 (복수 공고 비교 시)

공고 내용이 이미 대화 텍스트로 포함된 경우에도 동일하게 처리한다.

### 3-2. 사용자 정보

현재 대화에서 확인 가능한 정보를 우선 활용한다. 가능하면 반복 질문을 피하고, 불확실하면 짧게 재확인한다. 정보가 부족한 경우 부족함을 명시한 채 분석을 진행한다.

수집 항목:

- **학적 상태** (재학 / 휴학 / 졸업예정) → 지원 자격 판단
- **학년** (1~4학년 또는 대학원) → 지원 자격 판단
- **관심 기회 유형** (복수 선택: 대외활동 / 공모전 / 인턴십 / 장학금 / 교육 프로그램) → 목표 부합도 판단
- **목표** (복수 선택: 실무 경험 / 포트폴리오 강화 / 네트워크 형성 / 장학금 확보 / 스펙 쌓기 / 기타) → 목표 부합도 판단
- **보유 역량 또는 준비 상태** → 강점·보완점·체크리스트 판단
- **일정 및 활동 가능 조건** → 일정 충돌 가능성 판단
- **추가 설명** (선택): 특수 상황 등

## 4. 동작 흐름

### Step 1. 사용자 정보 확인

현재 대화에서 이미 확인된 항목은 다시 묻지 않는다. 아직 확인되지 않은 필수 항목(학적 상태, 학년, 목표)만 간단히 질문한다. 나머지 항목은 없어도 분석을 진행하되, 부족한 정보는 판단 근거에 명시한다. 없는 정보를 추측해 판단에 반영하지 않는다.

### Step 2. 문서 파싱

업로드된 공고 파일을 Upstage Document Parse API(Solar Document)로 읽어 마크다운 텍스트로 변환한다.

복수 파일은 순차 처리하며 진행 상황을 표시한다. 파싱에 실패하거나 내용이 인식되지 않으면 사용자에게 안내하고 해당 파일을 건너뛴다.

### Step 3. 핵심 정보 구조화

Step 2의 마크다운 텍스트를 Solar LLM에 전달해 아래 항목을 JSON으로 추출한다.

| 필드 | 설명 |
|---|---|
| `title` | 공고 제목 |
| `category` | 대외활동 / 공모전 / 인턴십 / 장학금 / 교육 프로그램 / 기타 |
| `host` | 주최 기관 |
| `deadline` | 지원 마감일 |
| `target` | 모집 대상 |
| `qualifications` | 지원 자격 요건 |
| `benefits` | 혜택 또는 지원 내용 |
| `required_docs` | 제출 서류 |
| `period` | 활동 기간 |
| `location` | 활동 장소 (온라인/오프라인) |
| `team_or_individual` | 팀/개인 구분 |
| `special_notes` | 기타 특이사항 |

문서에 없는 항목은 `null`로 반환한다. 절대 추측하지 않는다. 출력 시 `null` 항목은 "명시되지 않음" 또는 "추가 확인 필요"로 표현한다.

### Step 4. 적합도 분석

Step 3의 공고 정보와 사용자 정보를 Solar LLM에 전달해 적합도 판단 결과를 생성한다. 판단 일관성을 위해 소수의 퓨샷 예시를 별도 참조 파일(`scripts/analyze_opportunity.py`)에 둘 수 있다.

**출력 필드:**

| 필드 | 설명 |
|---|---|
| `verdict` | 바로 지원 가능 / 준비 후 지원 / 조건 확인 필요 / 비추천 |
| `verdict_reason` | 판단 근거 — 아래 4가지 관점을 모두 포함 |
| `summary_line` | 한 문장 요약 |
| `fit_score` | 1~10점 (전반적 부합도) |
| `strengths` | 사용자 강점 목록 |
| `gaps` | 부족한 점 또는 추가 확인 사항 목록 |
| `checklist` | 준비 항목 (item + priority: 필수/권장/선택) |
| `tips` | 공고 특화 조언 |

**판단 기준 및 원칙:**

판단은 공고 문서에 명시된 조건만을 기준으로 한다. 외부 검색이나 추측으로 조건을 보완하지 않는다. 불명확하거나 미명시된 핵심 조건은 "조건 확인 필요"로 처리한다.

`verdict_reason`은 아래 4가지 관점을 모두 포함해 서술한다:

1. **자격 요건 충족 여부** — 문서에 명시된 조건 기준
2. **사용자 관심/목표와의 부합도** — 관심 유형 및 목표 항목 기준
3. **일정·활동 가능 조건의 적합성** — 활동 기간, 온/오프라인 방식 등
4. **정보 부족 또는 불확실성** — 미명시 조건, 추가 확인 필요 사항

`verdict` 판정 기준:

- **바로 지원 가능**: 문서에 명시된 모든 자격 요건 충족, 즉시 준비 가능
- **준비 후 지원**: 1~2가지 부족하지만 준비하면 지원 가능
- **조건 확인 필요**: 핵심 조건이 불명확하거나 미명시 — 정보 부족에 대한 보수적 처리
- **비추천**: 자격 요건 미충족이 명확하거나 상황과 현저히 불일치

`fit_score`는 위 4가지 관점을 종합해 1~10점으로 산정한다. 자격 미충족이 명확하면 낮게, 핵심 조건이 불명확하면 중간값을 넘지 않도록 한다.

### Step 5. 준비 체크리스트 생성

분석 결과를 바탕으로 준비 항목을 우선순위별로 정리한다:

- **필수**: 지원에 반드시 필요한 항목
- **권장**: 경쟁력 강화에 도움이 되는 항목
- **선택**: 있으면 유리하지만 필수는 아닌 항목

### Step 6. 복수 공고 비교 및 우선순위 제안

공고가 2개 이상이면 각 공고를 개별 분석한 뒤, 사용자 목표 기준으로 우선순위와 장단점을 비교해 제안한다.

**우선순위 결정 원칙:**

- 기본 정렬 기준: fit_score (높을수록 우선)
- 추가 고려 요소: 자격 적합성, 역량 충족 여부, 준비 부담, 일정 적합성, 기대 효용
- 자격 미충족이 명확한 공고는 fit_score가 높더라도 불리하게 처리하며 이유를 명시한다
- 핵심 조건이 불명확한 공고는 해당 불확실성을 우선순위 설명에 언급한다

## 5. 출력 형식

### 5-1. 출력 구조

각 공고 분석 결과는 아래 템플릿을 따른다. 한국어로 출력한다.

표 형태 정보는 반드시 Markdown table로 출력한다. ASCII 박스, 박스 드로잉 문자, 공백 정렬은 사용하지 않는다. 섹션 제목은 Markdown heading(`##`, `###`, `####`)을 사용하고, 나열 항목은 `-` bullet list로 출력한다.

---

### [공고 제목] 분석 결과

#### 기본 정보

| 항목 | 내용 |
|---|---|
| 카테고리 | 대외활동 |
| 주최 | OO기관 |
| 마감일 | YYYY-MM-DD |
| 활동 기간 | X개월 |

#### 적합도 요약

| 항목 | 내용 |
|---|---|
| 지원 판단 | 바로 지원 가능 / 준비 후 지원 / 조건 확인 필요 / 비추천 |
| 목표 부합도 | 8/10 |
| 한줄 요약 | 핵심을 한 문장으로 압축 |

#### 판단 근거

- (자격 요건) 근거 1
- (목표 부합도) 근거 2
- (일정/활동 조건) 근거 3
- (정보 부족·불확실성) 근거 4 — 해당 사항 없으면 생략

#### 강점 및 보완점

| 구분 | 내용 |
|---|---|
| 유리한 점 | 사용자 강점 |
| 부족한 점 | 갭 또는 추가 확인 필요 사항 |

#### 준비 체크리스트

| 우선순위 | 준비 항목 | 설명 |
|---|---|---|
| 필수 | 자기소개서 작성 | 공고 요구사항에 맞춘 초안 준비 |
| 권장 | 관련 경험 정리 | 지원 동기와 연결할 사례 확보 |

#### 공고 특화 조언

- 이 공고에 맞는 지원 전략을 2~3개 bullet로 제시한다.

---

복수 공고인 경우 마지막에 우선순위 요약 표를 추가한다.

### 우선순위 요약

| 순위 | 공고명 | 추천 이유 |
|---|---|---|
| 1 | 공고명 A | 목표 부합도와 일정 적합성이 가장 높음 |
| 2 | 공고명 B | 준비 부담은 있지만 성장 경험 측면에서 유리함 |

### 5-2. 출력 원칙

- 단순 요약이 아니라 판단 중심으로 답한다.
- 문서에 없는 정보는 "명시되지 않음"으로 표시한다. 추측하지 않는다.
- 판단 근거에서 문서에 명시된 사실, 사용자가 제공한 정보, 불명확한 부분을 명시적으로 구분한다.
- 사용자가 바로 행동할 수 있도록 준비 항목을 구체적으로 제시한다.
- 필요하면 "직접 문의 권장", "공고 원문 재확인 필요" 안내를 포함한다.
- 마감이 24시간 이내로 임박한 경우 분석 결과 상단에 긴급 안내를 표시한다.
- 확실한 정보, 추정 가능 정보, 미확인 정보를 구분하고, 미확인 정보를 확정적으로 표현하지 않는다.
- 특정 학교·학과 배경이 자동으로 유리하거나 불리하게 반영되지 않는다.

## 6. 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 공고 1개만 업로드 | 비교/우선순위 생략, 단일 분석만 출력 |
| 마감일이 이미 지난 경우 | ⚠️ 참고용 분석임을 명시 후 진행 |
| 마감이 24시간 이내로 임박 | 🚨 긴급 안내 표시, 체크리스트를 즉시 행동 중심으로 재구성 |
| 공고 형식이 아닌 문서 | "공고 문서가 아닐 수 있음" 안내 |
| 자격 요건이 모호한 경우 | "조건 확인 필요" 판정, 직접 문의 권장 |
| 문서 품질 낮아 정보 불명확 | 한계를 함께 설명 |
| 외국어 공고 | Document Parse 처리 후 한국어로 분석 |
| 복수 파일 | 순차 처리 + 진행 상황 표시 (예: 1/3 파싱 완료) |
| 사용자 정보 부족 | 가능한 범위에서 분석, 부족한 정보로 인한 한계를 명시 |

## 7. 한계 및 주의사항

- 문서에 없는 자격 요건을 추측해서 단정하지 않는다.
- 외부 검색이나 외부 데이터 조회로 공고 조건을 보완하지 않는다.
- 불확실한 정보를 확정적으로 표현하지 않는다.
- 현재 대화에서 확인된 정보만 활용한다. 대화 외부의 정보를 저장하거나 기억하지 않는다.

## 8. 예시

**입력**

> "이 공고 나한테 맞아? 재학 중인 2학년이고, 파이썬 할 줄 알고, 여름방학 내내 시간 있어. 목표는 포트폴리오 강화야."
> (첨부: 2025 여름 AI 해커톤 PDF — 카카오 주최, 대학생 팀 2~4인, 온라인, 파이썬 우대, 마감 2025-07-15)

**출력**

### 2025 여름 AI 해커톤 분석 결과

#### 기본 정보

| 항목 | 내용 |
|---|---|
| 카테고리 | 공모전 |
| 주최 | 카카오 |
| 마감일 | 2025-07-15 |
| 활동 기간 | 3일 (2025-08-01 ~ 08-03) |

#### 적합도 요약

| 항목 | 내용 |
|---|---|
| 지원 판단 | 바로 지원 가능 |
| 목표 부합도 | 9/10 |
| 한줄 요약 | 자격·일정·역량 모두 충족하며 포트폴리오 목표와 잘 맞는 공모전이다. |

#### 준비 체크리스트

| 우선순위 | 준비 항목 |
|---|---|
| 필수 | 팀원 모집 (1~3명) |
| 필수 | 아이디어 기획서 초안 작성 (A4 3매) |
| 권장 | 머신러닝 관련 아이디어 구체화 |

## 9. 사용 시나리오

**시나리오 A: 복수 공고 비교 및 우선순위 제안**
> "공모전 PDF 3개 올릴게. 내 상황에서 어떤 걸 먼저 준비하는 게 나아? 1학년이고 포트폴리오 만들고 싶어."

→ 3개 공고를 각각 분석하고, 자격 요건 충족 여부·목표 부합도·준비 부담을 비교해 우선순위 표로 제안한다.

**시나리오 B: 자격 요건 기반 지원 가능 여부 판단**
> "이 장학금 공고, 내가 받을 수 있을까? 성적은 3.8이고 2학년 재학 중이야."

→ 공고에 명시된 성적·학년·학적 조건과 사용자 정보를 비교해 지원 가능 여부와 추가 확인 사항을 안내한다.

**시나리오 C: 준비 행동 제안 및 일정 적합성 판단**
> "여름방학에 할 수 있는 대외활동·인턴 공고 2개 올릴게. 어떤 걸 지원할 수 있고, 무엇부터 준비해야 해?"

→ 각 공고의 활동 기간·장소·제출 서류를 분석하고, 여름방학 일정과의 적합성을 판단한 뒤 준비 체크리스트를 제안한다.
